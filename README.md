# RaBbLE-Collective

**RaBbLE** is a unified ecosystem of projects sharing one entity, one visual language, and one purpose — to build a computing environment that is an expression of a coherent intelligence, not a collection of disconnected tools.

This repository is the coordination layer: the registry of all Collective members, the shared grimoire (documentation), and the scripts that wire it together.

---

## Why RaBbLE

Most computing environments are assembled — a distro here, a theme there, an AI assistant bolted on. RaBbLE is grown from a single root. Every project in the Collective shares the same entity (RaBbLE), the same palette (synthwave outrun), and the same philosophy (Low Entropy Directive). A user moving between the OS shell, the web interface, and the frontend feels one continuous environment — not a collection of products that happen to match.

The name is not an acronym. It is a character.

## Repository Structure

```
RaBbLE-Collective/
├── AGENT.md                  ← Agent entry point (LLM-agnostic)
├── CLAUDE.md / CODEX.md      ← Symlinks → AGENT.md (generated by setup.sh)
├── CONTEXT.md                ← Current epoch and active tracks
├── REFERENCES.md             ← Decisions, external links
├── README.md                 ← This file
│
├── grimoire/                 ← Collective documentation (the soul)
│   ├── distilled/            ← Generated agent reference files
│   └── *.md                  ← Canonical lore documents
│
├── registry/                 ← Member registration and epoch state
│   ├── manifests/            ← Per-project registry entries (YAML)
│   └── epochs/               ← Epoch definitions and archive
│
└── scripts/                  ← Collective tooling
    ├── setup.sh              ← Bootstrap: pull projects, wire symlinks
    ├── status.sh             ← Collective health dashboard
    ├── sync-grimoire.sh      ← Propagate grimoire updates to all members
    └── new-member.sh         ← Scaffold a new Collective project
```

## Getting Started

```bash
git clone git@github.com:markm1206/RaBbLE.git RaBbLE-Collective
cd RaBbLE-Collective
./scripts/setup.sh
```

`setup.sh` will:
- Create `CLAUDE.md` and `CODEX.md` as symlinks to `AGENT.md`
- Clone all registered Collective members
- Sync the core grimoire to each member project
- Report Collective health

After setup, check current state:

```bash
./scripts/status.sh
```

## Development Flow

The Collective uses **epoch-driven development** — bounded periods of intentional work with a shared intent and a defined exit condition. All projects move together.

Current epoch and active tracks: `CONTEXT.md`
Full epoch definition: `registry/epochs/current.epoch.yml`

Commits follow the Pulse Protocol:
```
spark ~ collective >> new capability manifested // %STATE_TAG%
transcribe ~ grimoire >> document updated
harmonize ~ registry >> entropy reduced
```

Full commit conventions: `grimoire/distilled/conventions.distilled.md`
